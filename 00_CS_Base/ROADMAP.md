# `00_CS_Base` - 計算機科学 詳細ロードマップ

このドキュメントは、すべての技術分野の根幹をなす「計算機科学（Computer Science）」の学習ロードマップです。

## 1. `Architecture` (コンピュータアーキテクチャ)

**目標:** コンピュータが内部でどのように動作しているかを理解し、低レイヤーの視点からパフォーマンスを意識したプログラミングができるようになる。

### Level 1: 基本構成要素の理解
- **学習内容:**
  - 論理回路（AND, OR, NOT, XOR）とブール代数
  - CPUの主要コンポーネント（ALU, 制御装置, レジスタ）
  - メモリ階層（レジスタ, L1/L2/L3キャッシュ, メインメモリ, ストレージ）
  - 命令セットアーキテクチャ（ISA）の基本
- **推奨リソース:**
  - **書籍:** 『コンピュータの構成と設計 (パターソン&ヘネシー)』 - 特に前半部分
  - **Web:** [Nand to Tetris](https://www.nand2tetris.org/) - 論理ゲートからコンピュータを構築していく素晴らしいプロジェクト
- **マイルストーン:**
  - Nand to Tetrisのプロジェクトを6章まで（アセンブラの完成）完了させる。

### Level 2: OSと連携した動作の理解
- **学習内容:**
  - 命令パイプライン、スーパースカラ
  - OSの役割（プロセスとスレッドの管理、メモリ管理、仮想記憶）
  - 割り込みと例外処理
- **推奨リソース:**
  - **書籍:** 『詳解Linuxカーネル』、『OS自作入門 (30日でできる!)』
- **マイルストーン:**
  - 『OS自作入門』を参考に、簡単なブートローダ、コンソール出力、割り込み処理を持つ小さなOSを自作する。

## 2. `Network` (ネットワーク)

**目標:** アプリケーション間の通信がどのようなプロトコルで実現されているかを深く理解し、ネットワーク関連の問題解決やパフォーマンスチューニングができるようになる。

### Level 1: TCP/IPの基礎
- **学習内容:**
  - OSI参照モデルとTCP/IPモデル
  - 主要プロトコル (IP, TCP, UDP, ICMP, DNS, HTTP, HTTPS)
- **推奨リソース:**
  - **書籍:** 『マスタリングTCP/IP 入門編』
  - **ツール:** Wireshark
- **マイルストーン:**
  - Wiresharkを使い、Webブラウジング時のパケット（DNSクエリ、TCP 3-wayハンドシェイク、HTTP/TLS通信）をキャプチャし、内容を説明できる。
  - Python等で簡単なHTTP/TCPサーバーとクライアントを実装する。

### Level 2: パフォーマンスと信頼性
- **学習内容:**
  - TCPの輻輳制御とフロー制御
  - TLS/SSLのハンドシェイクと暗号化の仕組み
  - HTTP/1.1, HTTP/2の多重化やヘッダ圧縮
- **推奨リソース:**
  - **書籍:** 『ハイパフォーマンスブラウザネットワーキング』(O'Reilly)
- **マイルストーン:**
  - 自作したHTTPサーバーにTLSを実装する（OpenSSLライブラリ等を利用）。
  - HTTP/2の主要な改善点を説明できる。

### Level 3: 最新プロトコルの探求
- **学習内容:**
  - HTTP/3とQUICプロトコル
- **推奨リソース:**
  - **Web:** CloudflareやGoogleの技術ブログ、関連するRFC
- **マイルストーン:**
  - QUICの利点（Head-of-Lineブロッキングの解決など）をTCPと比較して説明できる。
  - 既存のQUICライブラリを用いて簡単なクライアント・サーバーアプリケーションを作成する。

## 3. `Database_Internals` (データベース内部構造)

**目標:** データベースがどのようにデータを格納し、高速な読み書きやトランザクションを実現しているかを理解する。

### Level 1: 基本的なデータ構造とSQL
- **学習内容:**
  - リレーショナルモデルと正規化
  - SQLの実行計画
  - インデックスの基本（B-Tree, B+Tree）
- **推奨リソース:**
  - **書籍:** 『達人に学ぶDB設計 徹底指南書』、『SQLアンチパターン』
- **マイルストーン:**
  - 複雑なSQL（相関サブクエリ、ウィンドウ関数など）の実行計画を読み、インデックスの有効性を判断できる。
  - 簡単なB+Treeの構造をコードで表現してみる。

### Level 2: トランザクションと同時実行制御
- **学習内容:**
  - トランザクションのACID特性
  - 分離レベル（Read Uncommitted ~ Serializable）
  - 同時実行制御（MVCC: 多版同時実行制御）
  - データベースのリカバリ（WAL: Write-Ahead Logging）
- **推奨リソース:**
  - **書籍:** 『データ指向アプリケーションデザイン』
- **マイルストーン:**
  - MVCCがなぜノンブロッキングな読み取りを可能にするのか説明できる。
  - WALの役割と、クラッシュからの回復プロセスを説明できる。

### Level 3: 分散データベースと最新ストレージエンジン
- **学習内容:**
  - 分散システムの基本（CAP定理、一貫性モデル）
  - コンセンサスアルゴリズム（Paxos, Raft）
  - ストレージエンジン（LSM-Tree: Log-Structured Merge-Tree）
- **推奨リソース:**
  - **書籍:** 『データ指向アプリケーションデザイン』
  - **論文:** GoogleのSpanner, DynamoDB, Bigtableなどに関する論文
- **マイルストーン:**
  - Raftアルゴリズムのリーダー選出プロセスを説明できる。
  - B-TreeとLSM-Treeの書き込み・読み込みにおける性能特性の違いを説明できる。

## 4. `Algorithms_Advanced` (高度なアルゴリズム)

**目標:** AtCoder赤コーダーレベルに到達するために必要な、高度なアルゴリズムとデータ構造、そして数学的考察力を身につける。

### Phase 1: 基礎固め (AtCoder 緑/水色)
- **学習内容:**
  - 基本的なデータ構造（配列, スタック, キュー, ヒープ, ハッシュテーブル, 二分探索木）
  - 基本的なアルゴリズム（ソート, 二分探索, BFS, DFS）
  - 計算量（Big O記法）
- **推奨リソース:**
  - **書籍:** 『プログラミングコンテスト攻略のためのアルゴリズムとデータ構造』（螺旋本）
  - **Web:** AtCoder, LeetCode
- **マイルストーン:**
  - AtCoderのA, B, C問題を安定して時間内に完答できる。
  - LeetCodeのEasy問題を9割以上、Medium問題を5割以上解ける。

### Phase 2: 中級アルゴリズムの習得 (AtCoder 水色/青色)
- **学習内容:**
  - 動的計画法（DP）の基本パターン
  - グラフアルゴリズム（ダイクストラ法, ワーシャルーフロイド法, プリム法）
  - Union-Find木、セグメント木、Fenwickツリー（BIT）
- **推奨リソース:**
  - **書籍:** 『問題解決力を鍛える！アルゴリズムとデータ構造』（けんちょん本）
  - **Web:** AtCoder Problems (Typical 90)
- **マイルストーン:**
  - AtCoderのD問題を安定して解けるようになり、E問題にも挑戦する。
  - AtCoder 典型90問の★4までを自力で解ける。

### Phase 3: 上級アルゴリズムへの挑戦 (AtCoder 黄色/橙色)
- **学習内容:**
  - 高度なグラフアルゴリズム（最大フロー, 最小費用流）
  - 高度なDP（桁DP, 木DP, bitDP）
  - 数論（素数判定, mod演算, 繰り返し二乗法）
  - 計算幾何学の基本
- **推奨リソース:**
  - **Web:** AtCoderの過去問 (E, F)、TopCoderのチュートリアル
- **マイルストーン:**
  - AtCoderのE問題を安定して解き、F問題が解けることがある。
  - コンテストで黄色パフォーマンスを安定して出せる。

### Phase 4: 最上級への道 (AtCoder 赤色)
- **学習内容:**
  - 問題を解くための数学的考察力、抽象化能力
  - 高度なデータ構造（遅延評価セグメント木, 平衡二分探索木など）
  - マイナーだが強力なアルゴリズム（CHT, Mo's Algorithmなど）
- **推奨リソース:**
  - **Web:** Codeforces (Div1 E/F), yukicoder, AOJ, 最新の研究論文
- **マイルストーン:**
  - コンテストで橙色パフォーマンスを安定して出せる。
  - 過去問のF問題やG問題を解き、解説を読んで完全に理解する。
  - 自分なりのアルゴリズムライブラリを整備する。
