# `01_Data_Science_Expert` - データサイエンス 詳細ロードマップ

このドキュメントは、単にライブラリを使うだけでなく、理論を深く理解し、Kaggle Grandmasterレベルの知見を抽象化・応用して、ビジネス価値を創出する専門家になるためのロードマップです。

## 1. `Machine_Learning` (機械学習の理論と実装)

**目標:** Kaggleコンペティションで上位入賞できるレベルの実践力を身につけ、最新の論文を実装できる理論的基盤を構築する。

### Level 1: 基礎理論とテーブルデータ分析 (Kaggle Masterへの道)
- **学習内容:**
  - 線形回帰、ロジスティック回帰、決定木、サポートベクターマシン
  - アンサンブル学習（バギング, ランダムフォレスト, 勾配ブースティング - 特にLightGBMとXGBoost）
  - モデルの評価指標（AUC, F1-Score, RMSE, etc.）と交差検証（Cross-Validation）
  - 特徴量エンジニアリングの基本パターン
- **推奨リソース:**
  - **書籍:** 『Pythonではじめる機械学習』、『Kaggleで勝つデータ分析の技術』
  - **Web:** Kaggle Courses (Pandas, Intermediate Machine Learning)
- **マイルストーン:**
  - Kaggleのテーブルデータコンペティションに参加し、一通りのパイプライン（データ読み込み, 前処理, モデル学習, 予測, 提出）を自力で実装できる。
  - コンペティションでSilverメダルを獲得する。

### Level 2: 深層学習と主要フレームワーク
- **学習内容:**
  - ニューラルネットワークの基礎（順伝播と誤差逆伝播法）
  - 主要な活性化関数、損失関数、最適化アルゴリズム
  - CNN（畳み込みニューラルネットワーク）による画像認識
  - RNN, LSTM, GRUによる時系列・自然言語処理
  - Transformerの基本構造とSelf-Attention
- **推奨リソース:**
  - **書籍:** 『ゼロから作るDeep Learning』シリーズ、『PythonとKerasによるディープラーニング』
  - **Web:** Stanford CS231n (画像認識), CS224n (自然言語処理), PyTorch/TensorFlowの公式チュートリアル
- **マイルストーン:**
  - PyTorchまたはTensorFlowを使い、CIFAR-10（画像分類）やIMDB（感情分析）のような標準データセットでモデルをスクラッチから実装し、論文に近い精度を再現する。
  - Kaggleの画像または自然言語処理コンペに参加し、ベースラインを超えるスコアを出す。

### Level 3: 最新論文の実装と独自ライブラリ開発
- **学習内容:**
  - 最新のモデルアーキテクチャ（EfficientNet, Vision Transformer, BERTなど）
  - 生成モデル（GAN, VAE）、自己教師あり学習（SimCLRなど）
- **推奨リソース:**
  - **Web:** [Papers with Code](https://paperswithcode.com/), ArXiv, トップカンファレンス (NeurIPS, ICML, CVPR, ICLR, ACL, etc.) の論文
- **マイルストーン:**
  - 興味のある分野の最新論文を1つ選び、公開コードを参考にしながら再実装を試みる。
  - 自分が頻繁に使う前処理、モデル構造、学習パイプラインなどをまとめ、小さな個人用ライブラリとしてパッケージ化する。

## 2. `MLOps` (機械学習基盤)

**目標:** モデルを安定的に本番環境で運用し、ビジネスサイクルに組み込むための技術を習得する。

### Level 1: 実験管理とモデルのサービング
- **学習内容:**
  - 実験管理ツール（MLflow, Weights & Biases）によるパラメータ・結果の追跡
  - モデルのシリアライズ（pickle, joblib, ONNX）
  - REST APIによるモデルサービング（Flask, FastAPI）
- **推奨リソース:**
  - **Web:** MLflow, FastAPIの公式ドキュメント
- **マイルストーン:**
  - 自身が学習させたモデルをFastAPIでAPI化し、JSONリクエストを受け取って予測結果を返すWeb APIを構築できる。
  - MLflowを導入し、複数の実験のパラメータと評価指標をロギング・比較できる。

### Level 2: パイプラインの自動化とコンテナ化
- **学習内容:**
  - Dockerによる開発・実行環境のコンテナ化
  - CI/CD（GitHub Actionsなど）を用いたテストとデプロイの自動化
  - バッチ推論パイプラインの構築（Airflow, Kubeflow Pipelines, Prefectなど）
- **推奨リソース:**
  - **書籍:** 『ソフトウェアエンジニアのための機械学習デザインパターン』
  - **Web:** Docker, GitHub Actionsの公式ドキュメント
- **マイルストーン:**
  - 学習・推論環境をDockerコンテナ化し、どこでも再現可能にする。
  - GitHubリポジトリへのpushをトリガーに、自動でテストが実行され、モデルがAPIとしてデプロイされるCI/CDパイプラインを構築する。

### Level 3: モデル監視と高度なデプロイ戦略
- **学習内容:**
  - モデルの性能監視（Data Drift, Concept Driftの検知）
  - A/Bテスト、カナリアリリース、シャドウデプロイ
  - 推論サーバーのパフォーマンス最適化（ONNX Runtime, TensorRT）
- **推奨リソース:**
  - **書籍:** 『SREの探求』、『Reliable Machine Learning』
  - **Web:** Seldon Core, KServe (旧KFServing) のドキュメント
- **マイルストーン:**
  - 推論APIにモニタリング機構（Prometheus, Grafanaなど）を導入し、レイテンシやエラーレートを監視する。
  - 本番環境の入力データの分布変化を検知し、アラートを出す仕組みを実装する。

## 3. `Causal_Inference` (因果推論)

**目標:** 「相関」と「因果」を区別し、データからビジネスアクションに繋がる真の示唆を引き出す能力を身につける。

### Level 1: 基礎概念「相関と因果」
- **学習内容:**
  - ランダム化比較試験（RCT）の重要性
  - 交絡バイアス、セレクションバイアス
  - 構造的因果モデル（SCM）と有向非巡回グラフ（DAG）
  - バックドア基準、フロントドア基準
- **推奨リソース:**
  - **書籍:** 『効果検証入門』、『因果推論の科学 (The Book of Why)』
- **マイルストーン:**
  - ビジネス課題（例: クーポンを配布すべきか？）をDAGで表現し、交絡因子が何かを特定できる。
  - RCTがなぜ因果関係を正しく推定できるのかを説明できる。

### Level 2: 主要な手法の理解と実装
- **学習内容:**
  - 傾向スコア（Propensity Score）を用いた分析（マッチング, 逆確率重み付け法-IPW）
  - 回帰不連続デザイン（RDD）
  - 差分の差分法（DID）
  - 操作変数法（IV）
- **推奨リソース:**
  - **Web:** Pythonライブラリ `EconML`, `DoWhy` のドキュメント, [Causal Inference for The Brave and True](https://matheusfacure.github.io/python-causality-handbook/)
- **マイルストーン:**
  - 『効果検証入門』のサンプルデータを使い、傾向スコアを用いた分析（IPWなど）をPythonで再現・実装できる。
  - `DoWhy`ライブラリを使い、仮想的なビジネス課題に対して因果効果の推定を行う。

### Level 3: ビジネス応用と限界の理解
- **学習内容:**
  - Double Machine Learning
  - Meta-Learners (S-Learner, T-Learner, X-Learner)
  - 因果探索（LiNGAMなど）
  - 各手法の仮定と限界
- **推奨リソース:**
  - **Web:** Microsoft Research, Netflix, Uberなどの技術ブログにおける因果推論の応用事例
- **マイルストーン:**
  - 機械学習モデルを組み込んだ因果推論手法（Double MLなど）の概要を説明できる。
  - 実際のビジネスデータや公開データセットを使い、「なぜその結果になったのか」を因果推論を用いて分析し、非専門家にもわかるようにレポートを作成する。
