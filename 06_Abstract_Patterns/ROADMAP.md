# `06_Abstract_Patterns` - 抽象化とメタ学習 詳細ロードマップ

このドキュメントは、特定の技術や領域を超越し、ソフトウェア設計、意思決定、そして思考そのものの根底に流れるパターンを学び、より高い次元のエンジニアリングの知見を獲得するためのロードマップです。

## 1. `Design_Patterns` (デザインパターン)

**目標:** 再利用可能な設計の語彙を身につけ、先人たちが体系化したベストプラクティスを適用することで、柔軟で保守性の高いソフトウェアを効率的に構築する。

### Level 1: GoFデザインパターン
- **学習内容:**
  - GoF (Gang of Four) の23のデザインパターンの意図と構造
  - **生成パターン:** Singleton, Factory Method, Abstract Factory
  - **構造パターン:** Adapter, Decorator, Facade, Composite, Proxy
  - **振る舞いパターン:** Strategy, Observer, Template Method, Iterator, Command
- **推奨リソース:**
  - **書籍:** 『増補改訂版 Java言語で学ぶデザインパターン入門』、『Head Firstデザインパターン』
- **マイルストーン:**
  - 上記に挙げた特に重要な10〜15個のパターンを、自身の得意な言語で実際にコーディングして実装できる。
  - 既存のコードの中で、これらのパターンを適用してリファクタリングできる箇所を見つけ、実践できる。

### Level 2: エンタープライズアプリケーションアーキテクチャパターン
- **学習内容:**
  - レイヤ化アーキテクチャ（ドメイン、データソース、プレゼンテーション）
  - データソースパターン（Repository, Data Mapper, Active Record）
  - オブジェクトリレーショナルマッピング（ORM）のパターン
  - Webプレゼンテーションパターン（MVC, MVVM, MVP）
- **推奨リソース:**
  - **書籍:** 『Patterns of Enterprise Application Architecture』(PofEAA)
- **マイルストーン:**
  - 自身が過去に構築したWebアプリケーションが、どのアーキテクチャパターンに基づいているかを分析し、説明できる。
  - Repositoryパターンを実装し、ドメインロジックとデータアクセスロジックを明確に分離する。

### Level 3: 分散システム・クラウドデザインパターン
- **学習内容:**
  - **信頼性パターン:** サーキットブレーカー, タイムアウト, リトライ
  - **スケーラビリティパターン:** CQRS (コマンド・クエリ責務分離), イベントソーシング, シャーディング
  - **メッセージングパターン:** Publisher/Subscriber, メッセージキュー, SAGAパターン
- **推奨リソース:**
  - **書籍:** 『マイクロサービスアーキテクチャ』、『データ指向アプリケーションデザイン』
  - **Web:** Microsoft Azureの[Cloud Design Patterns](https://docs.microsoft.com/en-us/azure/architecture/patterns/)
- **マイルストーン:**
  - 簡単なサーキットブレーカーをライブラリを使わずに実装してみる。
  - 2つのサービスとメッセージキューを使い、SAGAパターンによる分散トランザクション（補償トランザクションを含む）をシミュレーションする。

## 2. `ADR` (Architecture Decision Records)

**目標:** 「なぜ」その技術やアーキテクチャを選択したのかという意思決定の背景とコンテキストを記録し、将来の自分やチームへの重要な知見として残す習慣を身につける。

### Level 1: ADRの基本を学ぶ
- **学習内容:**
  - ADRとは何か、なぜ重要なのか
  - ADRの基本的な構成要素（Title, Status, Context, Decision, Consequences）
- **推奨リソース:**
  - **Web:** Michael Nygardによる[オリジナルのブログ記事](https://cognitect.com/blog/2011/11/15/documenting-architecture-decisions)、Joel Parker Hendersonの[ADRリポジトリ](https://github.com/joelparkerhenderson/architecture-decision-record)
- **マイルストーン:**
  - 過去に自身が下した重要な技術的決定（例: 「WebフレームワークにReactを選んだ理由」）を思い出し、ADRのフォーマットで3つ記述してみる。

### Level 2: ADRを実践する
- **学習内容:**
  - ADRのライフサイクル（Proposed, Accepted, Supersededなど）
  - ADRを管理するためのツール（`adr-tools` CLIなど）
- **推奨リソース:**
  - **Web:** `adr-tools`のGitHubリポジトリ
- **マイルストーン:**
  - 新しい個人プロジェクトを開始し、重要な技術選択を行うたびにADRを記述する習慣をつける。
  - `adr-tools`を導入し、コマンドラインからADRの作成、一覧表示、リンク生成などを行えるようにする。

## 3. `Mental_Models` (メンタルモデル)

**目標:** 複雑な事象を理解し、より良い意思決定を行うための思考のフレームワークを学び、自身の「引き出し」を増やす。

### Level 1: 基本的な思考ツール
- **学習内容:**
  - **問題解決:** MECE, ロジックツリー, ピラミッド構造
  - **意思決定:** トレードオフ, 第二階思考（Second-Order Thinking）, 機会費用
  - **見積もり:** フェルミ推定
- **推奨リソース:**
  - **書籍:** 『考える技術・書く技術』、『イシューからはじめよ』、『The Great Mental Models』シリーズ
- **マイルストーン:**
  - 「日本の電柱の数は？」といったお題に対し、論理的な仮定を置いて桁違いでない数値を算出するフェルミ推定ができる。
  - ある技術的決定（例: 「マイクロサービス化する」）がもたらす短期的な影響（第一階）と、それによって引き起こされる長期的な影響（第二階）を書き出せる。

### Level 2: システム思考
- **学習内容:**
  - システム思考の基本（要素、関係性、機能/目的）
  - ストックとフロー
  - フィードバックループ（自己強化型ループ、バランス型ループ）
  - ボトルネック、制約条件、レバレッジポイント
- **推奨リソース:**
  - **書籍:** 『学習する組織』、『思考は現実化する』、『ザ・ゴール』
- **マイルストーン:**
  - 「技術的負債が増えていくプロセス」や「優秀なエンジニアが辞めていくプロセス」を、フィードバックループを用いて図で表現し、説明できる。
  - 自身が関わっているシステムの中で、最も小さな力で最も大きな変化を生む可能性のある「レバレッジポイント」はどこかを考察する。

### Level 3: 抽象化とアナロジー
- **学習内容:**
  - 優れた抽象化の原則（関心の分離、漏れのある抽象化の回避）
  - 異なる分野からのアナロジー思考（生物学、物理学、経済学などのモデルをソフトウェアに応用する）
- **推奨リソース:**
  - **書籍:** 『ゲーデル、エッシャー、バッハ』、『アンチフラジャイル』
- **マイルストーン:**
  - 自身がよく知る2つの異なる技術分野（例: TCPの輻輳制御と、アジャイル開発のWIP制限）の間にアナロジーを見出し、共通のパターンを抽出する。
  - 自身の専門分野の複雑な概念を、全くの初心者に優れたアナロジーを用いて説明する練習をする。
